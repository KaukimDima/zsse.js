    Реакт-роутер — основной способ делать клиентский роутинг в реакте.

    Роутинг в браузере работает с помощью DOM api, а именно, window.history.
    Это апи позволяет смотреть историю посещения текущей вкладки браузера, добавлять новые пути в историю вкладки, заменять текущий путь, а также переходить назад и вперед по истории.

    Реакт-роутер использует это апи и предоставляет возможность описать роутинг приложения используя компоненты, HOC и контекст. Работать с контекстом напрямую не нужно, вся работа будет сделана компонентами помощниками.


    1. Статический и динамический роутинг
    Начиная с версии 4, реакт-роутер использует динамический роутинг, который немного отличается от привычного статического, но позволяет делать больше и элегантнее, и больше вписывается в мир реакта.

    Пример статического роутинга — это когда в одном месте описаны все роуты приложения и методы/классы/функции, отвечающие за обработку роута(пример koa-router): 
    router
 .get('/', function*(next) {
   this.body = 'Hello World!';
 })
 .post('/users', function*(next) {
   // ...
 })
 .put('/users/:id', function*(next) {
   // ...
 })
 .del('/users/:id', function*(next) {
   // ...
 });
    Пример взят из koa-router, но так же работает любая система описания роутинга в django, ruby on rails, angular, ember, в любой системе на клиенте или сервере. Это классический пример описания роутинга приложения, и реакт-роутер называет это статическим роутингом.

    Динамический роутинг в противовес предлагает, во-первых, использовать роутинг в render методах компонент, во-вторых, менять его в зависимости от тех условий, которые сейчас необходимы приложению. Рендеринг происходит на этапе выполнения приложения, а не заранее, и это немного отличается в описании в сравнении со статическим роутингом.
    2. Работа с реакт-роутер
    Для установки нужно воспользоваться командой: 
    yarn add react-router-dom
    Также существует пакет react-router-native для работы на мобильных устройствах вместе с react-native, и react-router, который содержит общий код для этих библиотек.

    Первое, что нужно сделать — это обернуть корневой компонент с помощью BrowserRouter: 
    import {BrowserRouter} from 'react-router-dom';

ReactDOM.render(
 <BrowserRouter>
   <App />
 </BrowserRouter>,
 document.getElementById('root')
);
    Теперь можно использовать компоненты реакт-роутера в приложении.
    А вот и сам роутинг в приложении: 
    import React from 'react';
import {Link, Route} from 'react-router-dom';

const Home = () => <p>Home page</p>;
const About = () => <p>About page</p>;
const Topics = () => <p>Topics page</p>;

const App = () => (
 <div>
   <ul>
     <li><Link to="/">Home</Link></li>
     <li><Link to="/about">About</Link></li>
     <li><Link to="/topics">Topics</Link></li>
   </ul>

   <hr />

   <Route exact path="/" component={Home} />
   <Route path="/about" component={About} />
   <Route path="/topics" component={Topics} />
 </div>
);

export default App;
    Обратите внимание, компоненты Route находятся рядом с обычными компонентами. Принцип работы реакт-роутера прост, там где есть соответствие пути браузерной строки и пути описанным в Route, там и происходит рендер компоненты. Реакт-роутер заменяет компоненту Route, у которой путь совпадает, на компоненту, которую нужно отрендерить компонентой Route. Те компоненты, путь которых не совпадает, просто не рендерятся, компонента Route в таком случае возвращает null, и содержимое не отражается.

    Для работы с переходами по страницам используется компонента Link, которая превращается в тег a, но реакт-роутер обрабатывает переход так, чтобы не было перезагрузки страницы. Роутер ловит событие клика на ссылку, отменят его, и делает push в window.history, браузер переходит на новый путь без перезагрузки страницы.

    Компоненты, которые возвращает компонент Route получают в props дополнительный параметры: 
    match = {
 isExact: true,
 params: {},
 path: '/topics',
 url: '/topics'
}

location = {
 hash: "",
 pathname: "/topics",
 search: "",
 state: {}
}
    и history.

    Match содержит данные по соответствию пути и роута и дополнительные параметры.

    IsExact говорит о том, что путь в браузерной строке полностью соответствует роуту.

    В params находятся дополнительные ключи, которые можно спарсить из роута, если путь использует регулярное выражение или /topics/:id.

    Path — это путь с которым проводили сравнение. Полезен для создания вложенных Route.

    Url — это полный путь, полезен для создания вложенных Link.


    Если использовать регулярные выражения в Route, например, /topics/:id, в path будет /topics/:id, а в url /topics/1.

    Location объясняет, где приложение сейчас, какие значения в адресной строке. Также объекты Location могут быть переданы в Link, и тогда навигацию можно делать не только по соответствию Url, но и по истории браузера, например, в state можно указать место, откуда был совершен переход, и сделать свою кнопку назад для возврата на предыдущую страницу.

    History — это объект, который использует библиотеку history для удобной работы с сессией и историей переходов. Также реакт-роутер может работать с якорными переходами, когда url меняется не целиком, а только после якоря #.

    History объект обычно содержит в себе следующие ключи:

    length — длина истории, количество location в стеке истории.

    action — текущее действие, выполненное с историей: PUSH, REPLACE или POP. Соответствует переходу на страницу, замене текущей страницы и переходу назад.

    location — текущая локация, соответствует location, переданному отдельно через prop.

    push — функция, с помощью которой происходит переход по истории.

    replace — функция для замены текущего пути.

    go(n) — функция для указания номера пути в стеке истории путей.

    goBack — функция для перехода назад (go(-1)).

    goForward — функция для перехода вперед(go(1)).

    block(promt) — функция для остановки перехода пользователя, которая вызовет окно confirm с просьбой не покидать страницу ("Вы уверены что хотите закрыть страницу?").
    3. Вложенные роуты 
    import React from 'react';
import {Link, Route} from 'react-router-dom';

const Home = () => <p>Home page</p>;
const Cats = ({match}) => <p>Cats page {match.params.id}</p>;
const Hobbies = ({match}) => (
 <div>
   <p>Hobbies page {match.params.id}</p>
   <Link to={`${match.url}/cats/1`}>cats 1</Link>
   <Link to={`${match.url}/cats/2`}>cats 2</Link>
   <Route path={`${match.path}/cats/:id`} component={Cats} />
 </div>
);
const About = ({match}) => (
 <div>
   <p>About page</p>
   <Link to={`${match.url}/hobbies/1`}>Hobbies 1</Link>
   <Link to={`${match.url}/hobbies/2`}>Hobbies 2</Link>
   <Link to={`${match.url}/hobbies/3`}>Hobbies 3</Link>
   <hr />
   <Route path={`${match.path}/hobbies/:id`} component={Hobbies} />
 </div>
);
const App = () => (
 <div>
   <Link to="/">Home</Link>
   <Link to="/about">About</Link>
   <hr />
   <Route exact path="/" component={Home} /> 
   <Route path="/about" component={About} />
 </div>
);
export default App;
    Вложенные роуты работают так же, как и ожидалось. Route размещаются в потомках, и если роут соответствует браузерной строке — роут рендерит свой компонент. Стоит обратить внимание на формирование ссылок и роутов. Для вложенных компонент идёт конкатенация с текущем роутом, в случае компоненты Hobbies ссылки в match будут: /hobbies/1, а ссылки в линк будут: /hobbies/1/cats/1. С роутами все тоже самое, но ссылки во вложенных роутах уже будут такими: /hobbies/:id/cats/:id.
    4. Switch и редиректы
    Для работы роутинга компоненты Route обычно оборачивают в компонент Switch. Его преимущество в том, что он проверяет, какой из детей соответствует текущему location и выполняет рендеринг только этой компоненты. По этой причине в DOM не попадут роуты, которые возвращают null, их рендеринг не будет вызван и в целом они не будут отработаны, что экономит клиентам процессор :-)

    Также это позволяет обрабатывать ошибки 404, и делать редиректы с разных путей на разные пути.
    Пример:
    <Switch>
 <Route path="/" exact component={Home} />
 <Redirect from="/old-match" to="/will-match" />
 <Route path="/will-match" component={WillMatch} />
 <Route component={NoMatch} />
</Switch>;
    В данном случае редирект будет только с пути old-match, а если не один из роутов не совпадет — выполнится Route без пути, и исполнится компонент NoMatch.

    Компонент Switch последовательно смотрит детей и сравнивает их path и флаги exact и strict.
    5. API react-router
    BrowserRouter

    BrowserRouter — это корневой элемент, который сообщает через контекст о текущем пути, использует history API для работы с браузерной строкой.

    props:

    • basename: string. Базовый путь для ваших путей, полезно, если на сайте частично роутингом занимается другое приложение и необходимо создать роутинг только для определенного сегмента url.

    • getUserConfirmation: func. Функция для создания подтверждения на каждый переход по истории.

    • forceRefresh: bool. Если значение true реакт роутер будет делать переходы с перезагрузкой страницы. Полезно, когда браузер не поддерживает history API.

    • keyLength: number. Для поля key, которое помещается в каждый объект location. PureComponent таким образом может понять, что мы обновили страницу, но путь остался прежнем.

    HashRouter

    HashRouter — аналог BrowserRouter, но вся работа происходит с якорными ссылками.

    props:

    • basename: string

    • getUserConfirmation: func

    • hashType: string. Доступно 3 опции, по умолчанию slash:

      • slash — #/about/hobbies

      • noslash — #about/hobbies

      • hashbang — #!/about/hobbies

    MemoryRouter

    Аналог BrowserRouter, но адресная строка такого роутера будет в памяти, полезно для написания тестов.


    props:

    • initialEntries: array. Список location в истории роутинга.

    • initialIndex: number. Индекс, указывающий на значение в initialEntries.

    • getUserConfirmation: func.

    • keyLength: number.

    StaticRouter

    Роутер, который никогда не меняет location. Полезен для server-side-rendering на стороне сервера, когда нет переходов и только одна точка входа.

    Также этот роутер может быть полезен в тестах.

    props:

    • basename: string

    • location: string. URL который получает сервер, req.url на node серверах.

    • location: object. Объект локации.

    • context: object. Простой объект js, можно использовать пустой. Компоненты могут складывать туда данные во время рендеринг, нужен для server-side-rendering.

    Router

    Компонент Router является предком компонентов BrowserRouter, HashRouter, MemoryRouter, NativeRouter и StaticRouter. Самое популярное использование интерфейса этого компонента — хранение переходов на странице с системой управления состояниями, типа redux или mobx.

    props:

    history: object. Объект истории для менеджмента истории.
    Link

    Основной способ создавать ссылки с помощью react-router.

    props:

    • to: string. Путь, на который нужно сделать переход.

    • to: object. Объект location для перехода.

    • replace: bool. Если установлен в true текущий location будет замещен новым.

    NavLink

    Специальная версия Link, имеет дополнительный атрибут указания css-класса или стилей, которые будет присвоены если ссылка NavLink совпадает с путем в браузере.

    props:

    • activeClassName: string. Класс, который будет присвоен элементу, если ссылка совпадает. Значение будет добавлено к className.

    • activeStyle: object. Стиль, который будет присвоен элементу, если совпадает.

    • exact: bool. Если значение true, то будет проверка на точное совпадение, а не на вхождение.

    • strict: bool. Если значение true, конечный слэш будет учтен при сравнении путей.

    • isActive: func. Функция для добавления дополнительной логики для определения, совпадают ли пути или нет. Аргументами функции будут объекты match и location.

    • location: object. Функция в isActive сравнивает текущий location из истории браузера. Если нужно сравнивать с другим location, то передать его можно в этой переменной.
    Prompt

    Компонент, который будет выводить окно с вопросом о согласии с переходом.

    props:

    • message: string. Строка, которая будет показа в браузере.

    • message: func. Функция, которая будет вызвана с передачей нового объекта location. Функция должна возвращать строку. 
      <Prompt 
message={location => `Are you sure you want to go to ${location.pathname}?`} 
/>;
      • when: bool. Если передать true, будет вызвано окно при переходе, если false — то нет.

        Redirect

        Компонент который производит редирект на новый адрес каждый раз когда его рендерят. Новый путь заменит текущий.

        props:

        • to: string. Путь, на который нужно сделать переход.

        • to: object. Объект location для перехода.

        • push: bool. Если true, переход будет добавлен, и не заменит текущий путь.

        • from: string. Путь с которого нужно сделать редирект. Используется в Switch.
          Route

          Компонент, который рендерит своё содержимое в том случае, когда происходит совпадение адреса. Если нет совпадения — возвращает null.

          Route позволяет 3 способами указать то, что нужно отрендерить: component, render и children. Каждый из этих параметров полезен по своему. Возможно использовать одновременно только один из этих способов. Самый практичный из них — component.

          props:

          • component: Component. Реакт компонент, который будет отрендерен, когда путь роута совпадет. Компоненту будет переданы параметры location, match и history. При передаче в этот параметр функции, а не компонент(Component, PureComponent), будет заново произведен ререндер, каждый раз когда рендериться родитель. Происходит unmounting и mounting каждый раз. Если использовать component, то будет только update компоненты.
            //Для работы с inline rendering, то есть для таких случаев, как:

() => <p>Home page</p>

//нужно использовать параметр render.
            • render: func. Этот параметр позволяет добавлять inline rendering, который будет исполнен без unmounting и mounting каждый раз, когда родитель производит render. Функции, как и в случае метода component, будут переданы параметры location, match и history.

            • children: func. Существуют ситуации, когда рендер необходимо произвести вне зависимости от того, совпал путь или нет. В таком случае параметр children может помочь. Children работает так же как и render, если путь не совпадает, аргумент рендер функции match будет null. Такое поведение может быть полезно для работы с анимациями при переходе с одной страницы на другую.

            • path: string. Путь, по которому Route будет отрисовывать свое содержимое.

            • exact: bool. Если установлен в true, содержимое будет отрендерено только в случае полного совпадения. Полезно, когда используется корневой путь — /. Если указан false, то вхождение path в путь адресной строки будет вызывать рендер содержимого.

            • strict: bool. Когда указан true, слэш на конце строки будет учтён при сравнении.

            • location: object. Компонент Route будет сравнивать объект текущий location с объектом переданным в location компоненты. Это бывает нужно для сравнения location с текущем location, для проверки входит ли он в историю путей страницы.

              Switch

              Рендерит первый ребенок Route или Redirect, который совпадает с текущем location. Switch рендерит только одного ребенка, остальные дети не возвращают даже null.

              props:

              • location: object. Объект локации который будет использован в сравнении вместо текущего location.

                withRouter

                Это HOC, необходим для работы компонент, когда их родитель блокирует обновление contex t(своя проверка shouldComponentUpdate или PureComponent).

                Этот HOC, добавляет в props оборачиваемой компоненты объекты match, location и history. Также помогает при работе react-redux при оборачивании компоненты в HOC connect нужно использовать withRouter оборачивая снаружи connect — withRouter(connect()(App)).
